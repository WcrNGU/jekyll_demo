早就知道github这个平台，也曾经尝试过，最终因为英语水平太差看到全英文的界面就头疼最后放弃了。最近闲着没事研究了一番，突然发现并不需要多高的英语水平就能满足基本的使用，所以开始慢慢了解这个平台。

从一开始一头雾水，到慢慢明白什么是库(repository)，创建分支(branch),提交修改(commit)，合并分支(merge),再到后来大概弄懂了本地库与远程库的交互，先使用可视化的客户端到现在客户端和命令行交替使用(主要原因还是有些操作用命令行实现不了或者有错误)，然后又通过网上的教程学会了用jekyll构建静态的github page博客，之间出现过无数的问题，搞得我手忙脚乱，到最后终于都搞定了，现在决定把过程中遇到的坑记录下来。

  • 在windows下创建记事本并编辑html代码，保存为html格式并上传到个github，不久收到了github的邮件说页面创建失败，原因是因为不是utf-8码，于是百度了一下，找到了解决办法，把文件打开然后另存为同名文件覆盖，保存的时候在另存为界面选择编码方式为utf-8。上传后通过验证，本以为万事大吉，一打开页面傻眼了，页面所有的代码都是以字符方式呈现。试过很多办法都不好使，直到在jekyll官网看到了这句话"如果你使用 UTF-8 编码，那么在你的文件中一定不要出现 BOM 头字符，否则你会碰上非常糟糕的事情，尤其当你在 Windows 上使用 Jekyll 的时候。"，开始我并不在意，然后我在github打开html文件的时候，开头出现了一个奇怪的点，鼠标停在这个点上的时候，出现了几个字符，正是如此我才找到了问题的关键。通过搜索引擎找到了解决的办法，在网上下载了notepad++，里面有一个功能就是转为utf-8无BOM编码格式。至此我的jekyll博客终于正确显示。

  • 博客搭建完成后第一件事自然是赶紧写一篇博客记录一下，然而新的问题又出现了。上传新的文章后只有日期，没有超链接。于是我猜想还是编码方式的问题，于是做了三组对照实验
    1.记事本创建，用notepad++改为无bom的utf-8
    2.notepad++创建无bom的utf-8
    3.用github网页创建，然而三组实验全部失败。但是有心栽花花不成，无心插柳柳成荫，在创建这三个文档的时候我分别命名2017-1-1，2017-1-2，2017-1-3，但是最后的日期显示的却都是三月19日，于是我确定了肯定是jekyll的头部识别不了，于是我开始改，最后发现一条规律，“layout: 布局方法”冒号后面必须有一个空格，没有空格就识别不了，title也一样，至此，终于能顺利的发博客了。
